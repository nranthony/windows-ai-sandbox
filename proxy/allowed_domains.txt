# =============================================================================
# Outbound domain allowlist for windows-ai-sandbox
# =============================================================================
# One domain per line. A leading dot (e.g. .github.com) matches all subdomains.
# Comments use `#` at the START of a line. INLINE comments after a domain
# WILL BREAK Squid — `dstdomain` treats the whole line (including `#` and
# any trailing text) as one hostname string. Keep notes on their own lines.
#
# Three lifecycle tiers, top to bottom:
#   - ALWAYS ON: every profile needs these. Never comment out.
#   - PROJECT-PERSISTENT: the dev/ML stack. This repo's primary workload is
#     CUDA/ML dev, so package registries + CDNs are uncommented by default
#     (unlike macolima, where they live in PLANNING-MODE and are commented).
#     Comment a block out only for fully autonomous agent runs that should
#     not be able to install or fetch anything.
#   - PLANNING-MODE: short-lived installs you want to gate manually. Empty
#     today — most of what was here lives in PROJECT-PERSISTENT to match
#     this repo's policy. Add commented blocks here if you want a domain
#     reachable only while you're driving the shell.
#
# Hot-reload after edits:
#   COMPOSE_PROJECT_NAME=ai-sandbox-<profile> PROFILE=<profile> \
#     docker compose restart egress-proxy
#
# Blocks are tagged like `[name]` for grep — e.g. `grep "\[claude\]" allowed_domains.txt`.
# To bulk-toggle a block: select its lines and `s/^# //` (uncomment) or
# `s/^/# /` (recomment). Blank lines inside blocks are intentional visual
# chunking — Squid ignores them.
# =============================================================================
